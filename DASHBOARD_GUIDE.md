# Dashboard Quick Guide

## What this dashboard does
The dashboard in `interactive_dashboard.py` is a frame-level viewer for Switzerland vs Germany that combines:
- on-pitch context,
- model flow timeline,
- event jump points,
- and per-frame decision metrics.

It helps you inspect where the model thinks capacity (threat potential) rises or drops and compare taken actions vs better alternatives.

## Run the dashboard
From this folder:

```bash
pip install -r requirements.txt
python interactive_dashboard.py
```

Then open the local Dash URL shown in the terminal (usually `http://127.0.0.1:8050`).

## If cache is missing
The dashboard works best with a precomputed cache file at `outputs/full_game_max_flow_cache.csv`.

Generate it once with:

```bash
python "network flow visualization v3.py" --mode precompute --output-path outputs/full_game_max_flow_cache.csv
```

Optional quick test on a smaller frame range:

```bash
python "network flow visualization v3.py" --mode precompute --precompute-start-frame 9560 --precompute-end-frame 9700 --output-path outputs/full_game_max_flow_cache.csv
```

## How to use it
1. Use the time slider or Play button to move through the match.
2. Click rows in the Events table to jump to key moments.
3. Use the flow chart to see control swings over time.
4. Check the options/metrics panel for model-supported decision context.

## Small code map
- `interactive_dashboard.py`
  - Main Dash app and callbacks.
  - Builds timeline, score, event table, and visuals.
- `network flow visualization v3.py`
  - Core flow/capacity modeling logic and precompute pipeline.

- `match.json`
